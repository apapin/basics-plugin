# Preferences

- When I say "ok", "let's go", "go for it", or "go for option N" — proceed immediately, no confirmation needed.
- When presenting options, number them clearly so I can reply with just a number.
- Be direct and concise. Lead with the action, not the reasoning.
- At the end of long sessions with significant decisions, offer to update the project MEMORY.md.

# Conventions

- Use mise for tool version management.
- Use pnpm (not npm or yarn) for Node.js projects.
- Use conventional commits with scope: `<type>(<scope>): <description>`
- The top-level `yuma/` directory is a wrapper — each subdirectory is its own git repo. Always run git commands from within the actual repo.
